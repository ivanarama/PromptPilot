🏗️ Как это реализовать (реально рабочая архитектура)
1. База задач

Например:

{
  "id": 1,
  "prompt": "...",
  "status": "pending",
  "run_at": "2026-03-25T03:00:00",
  "retries": 0
}
2. Воркер (самое важное)

Скрипт, который:

Берёт задачу
Запускает claude CLI
Если ошибка лимита:
парсит текст ошибки
ставит retry через X минут
3. Детект лимита (ключевая фишка)

Например:

claude code "prompt" 2>&1

И ищешь:

rate limit
quota exceeded

👉 если нашёл:

откладываешь задачу
4. Планировщик

Можно:

cron (каждую минуту)
или бесконечный loop
5. Очередь

Минимально:

SQLite

Лучше:

Redis + BullMQ
🔥 Минимальный MVP (очень просто)

На Python:

import time
import subprocess

while True:
    task = get_next_task()

    if not task:
        time.sleep(10)
        continue

    result = subprocess.run(
        ["claude", "code", task["prompt"]],
        capture_output=True,
        text=True
    )

    if "rate limit" in result.stderr.lower():
        reschedule(task, delay=3600)
    else:
        mark_done(task)

  CLI команды

  python -m promptpilot.cli add "твой промпт"          # добавить задачу
  python -m promptpilot.cli add "..." -p 1 -a "2026-03-25T03:00"  # с приоритетом и расписанием
  python -m promptpilot.cli add -f prompts.txt          # из файла
  python -m promptpilot.cli list                        # список задач
  python -m promptpilot.cli status 1                    # детали задачи
  python -m promptpilot.cli cancel 1                    # отменить
  python -m promptpilot.cli stats                       # статистика
  python -m promptpilot.cli worker                      # запустить воркер
  python -m promptpilot.cli server                      # запустить веб-UI (http://127.0.0.1:8420)

  Ключевые фичи

  - Очередь задач с приоритетами (1-10) и расписанием
  - Автодетект rate limit — парсит ошибки Claude CLI
  - Exponential backoff — автоматический retry с увеличением задержки (60s -> 1h)
  - Crash recovery — при перезапуске воркера "зависшие" задачи возвращаются в очередь
  - Веб-интерфейс — dark theme, фильтры по статусу, авто-обновление каждые 5 сек, Ctrl+Enter для отправки
  - SQLite — без внешних зависимостей, данные в ~/.promptpilot/

  Для запуска нужны два процесса: worker (выполняет задачи) и server (веб-UI). Можешь попробовать прямо сейчас!
